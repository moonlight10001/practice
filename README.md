# practice
Маршрутизатор (C#)
Маршрут представляет из себя строку, состоящую из сегментов, разделенных символом /
Например: /foo/ /foo/bar/ /foo/bar/baz/ /foo/bar/1/ /foo/bar/2/

1. Сегмент - строка, находящаяся между символами /
Например: в маршруте /foo/bar/ , foo и bar- сегменты
2. Соответствие маршрута вызываемому обработчику задается шаблоном
3.  Шаблон - строка, задающая маршрут и состоящая из статических и динамических сегментов, разделенных символом /
Например: шаблон, задающий маршрут, который состоит только из статических сегментов /foo/bar/
шаблон, задающий маршрут, который состоит из статических и динамических сегментов: /foo/bar/{p:int}/
4. Динамический сегмент- сегмент, который содержит переменную, заданную правилом
Например: {p:int} задает правило, по которому сегмент маршрута содержит переменную с названием p типа int
5. Каждый маршрут однозначно сопоставляется с вызываемым делегатом при помощи шаблона
Например: маршрут, заданный шаблоном /foo/bar/ может быть сопоставлен с делегатом Action action = () => { };
маршрут, заданный шаблоном /foo/bar/{p:int}/ может быть сопоставлен с делегатом Action<int> action = (p) => { };

Реализация маршрутизатора должна уметь регистрировать маршруты и вызывать их обработчики.
Маршруты должны поддерживать переменные сегменты разных типов, например string, int, guid, float, datetime (поддержка всех типов не обязательна, достаточно несколько)
